StudentFlow is a web-based system designed to streamline the management of academic requests between students, lecturers, and administrative staff. The system provides an efficient, transparent, and role-based workflow for submitting, tracking, and processing student requests within an academic institution.

## Overview
The platform enables students to submit various academic requests and monitor their status in real time, while academic and administrative staff can review, manage, and respond to requests through dedicated dashboards. The system improves communication, reduces manual workload, and ensures structured request handling across the institution.

## Key Features
- Role-based access control for students, lecturers, and administrative staff  
- Submission and management of academic requests  
- Real-time request status tracking  
- Advanced filtering and search capabilities  
- Secure authentication and authorization  
- Responsive and user-friendly interface  

## System Architecture
- Backend implemented using **Python and Django**, following RESTful API principles  
- Frontend developed with **React**, providing dynamic dashboards and interactive user experience  
- Secure authentication using **JWT**  
- Database design supporting scalable and reliable data management  

## Technologies Used
- **Backend:** Python, Django, Django REST Framework  
- **Frontend:** React, JavaScript  
- **Database:** SQL-based relational database  
- **Authentication:** JWT  
- **DevOps & Tools:** Git, GitHub  

## Project Goals
- Improve efficiency in handling academic requests  
- Provide transparency and accountability throughout the request lifecycle  
- Enable scalable and maintainable system architecture  
- Deliver a clean and intuitive user experience
